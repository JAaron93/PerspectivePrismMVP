Popup States Test - Perspective Prism Extension

Test Overview

This document provides a comprehensive testing guide for all popup states implemented in task 9.2. Each state should be manually verified to ensure correct behavior.

1. Not on YouTube State Manual Test Required

Trigger: Open popup while on any non-YouTube page

Expected Display:

  • Icon: ℹ️ (info icon)
  • Message: "Navigate to a YouTube video"
  • Details: "to analyze"
  • Status class: info (blue background)
  • Progress bar: Hidden
  • Clear Cache button: Enabled

Test Steps:

  • Navigate to google.com or any non-YouTube site
  • Click the extension icon to open popup
  • Verify the status message and styling
  • Verify cache stats are displayed
  • Verify both buttons are visible and functional

2. On YouTube - Idle State Manual Test Required

Trigger: Open popup on a YouTube video page with no active analysis

Expected Display:

  • Icon: ✓ (checkmark)
  • Message: "Ready to analyze"
  • Details: "Video: [11-character video ID]"
  • Status class: success (green background)
  • Progress bar: Hidden
  • Clear Cache button: Enabled

Test Steps:

  • Navigate to any YouTube video (e.g., youtube.com/watch?v=dQw4w9WgXcQ)
  • Click the extension icon to open popup
  • Verify the video ID is displayed correctly
  • Verify the success styling (green border)
  • Verify both buttons are enabled

3. Analysis in Progress State Manual Test Required

Trigger: Open popup while analysis is running

Expected Display:

  • Icon: ⏳ (hourglass)
  • Message: "Analyzing video..."
  • Details: Empty
  • Status class: info (blue background)
  • Progress bar: Visible with percentage (e.g., "80%")
  • Clear Cache button: Disabled

Test Steps:

  • Navigate to a YouTube video
  • Click the "Analyze" button in the content script
  • Immediately open the popup
  • Verify the progress bar is visible and animating
  • Verify Clear Cache button is disabled
  • Keep popup open and watch it update as analysis progresses
Note: The popup polls for status updates every 2 seconds, so you should see the progress bar update in real-time.

4. Analysis Complete (Fresh) State Manual Test Required

Trigger: Open popup after a fresh analysis completes

Expected Display:

  • Icon: ✓ (checkmark)
  • Message: "Analysis complete"
  • Details: "Found X claim(s)\n(Just analyzed)"
  • Status class: success (green background)
  • Progress bar: Hidden
  • Clear Cache button: Enabled

Test Steps:

  • Navigate to a YouTube video that hasn't been analyzed
  • Click the "Analyze" button
  • Wait for analysis to complete
  • Open the popup
  • Verify "Just analyzed" is shown in details
  • Verify the claim count is correct

5. Analysis Complete (Cached) State Manual Test Required

Trigger: Open popup on a video with cached results

Expected Display:

  • Icon: ✓ (checkmark)
  • Message: "Showing cached results"
  • Details: "Found X claim(s)\n(Analyzed Y time ago)"
  • Status class: success (green background)
  • Progress bar: Hidden
  • Clear Cache button: Enabled

Test Steps:

  • Analyze a video and wait for completion
  • Navigate away and return to the same video
  • Open the popup
  • Verify "Showing cached results" message
  • Verify time ago is displayed (e.g., "2 minutes ago", "1 hour ago")
  • Wait a few minutes and reopen popup to verify time updates
Note: Time ago formats: "just now", "X minute(s) ago", "X hour(s) ago", "X day(s) ago"

6. Error State Manual Test Required

Trigger: Open popup after an analysis error

Expected Display:

  • Icon: ⚠️ (warning)
  • Message: Error message (e.g., "Analysis failed")
  • Details: Error details or "Check settings and try again."
  • Status class: error (red background)
  • Progress bar: Hidden
  • Clear Cache button: Enabled

Test Steps:

  • Configure an invalid backend URL in settings
  • Navigate to a YouTube video
  • Click the "Analyze" button
  • Wait for the error to occur
  • Open the popup
  • Verify error message is displayed with red styling
  • Verify error details provide helpful information
Alternative Test: Stop the backend server and try to analyze a video to trigger a connection error.

7. Not Configured State Manual Test Required

Trigger: Open popup when backend URL is not configured

Expected Display:

  • Icon: ⚙️ (gear)
  • Message: "Setup required"
  • Details: "Configure backend URL to start analyzing videos."
  • Status class: warning (orange background)
  • Progress bar: Hidden
  • Clear Cache button: Enabled

Test Steps:

  • Clear extension storage (chrome://extensions → Details → Clear storage)
  • Or manually remove the backend URL from settings
  • Navigate to a YouTube video
  • Open the popup
  • Verify "Setup required" message with orange styling
  • Click "Open Settings" and verify it opens the options page

8. Real-time Updates Test Manual Test Required

Purpose: Verify popup updates in real-time during analysis

Test Steps:

  • Navigate to a YouTube video
  • Open the popup (keep it open)
  • In the YouTube tab, click the "Analyze" button
  • Watch the popup update from "Idle" → "In Progress" → "Complete"
  • Verify the progress bar appears and updates
  • Verify the final state shows correct claim count
Note: The popup polls every 2 seconds and also listens for real-time messages from the background worker.

9. Cache Statistics Test Manual Test Required

Purpose: Verify cache stats display correctly in all states

Test Steps:

  • Open popup and note the cache stats (e.g., "5 videos (1.2 MB)")
  • Analyze a new video
  • Reopen popup and verify cache count increased
  • Click "Clear Cache" button
  • Verify cache stats show "0 videos (0.00 MB)"
  • Verify success message is displayed

10. Button Functionality Test Manual Test Required

Purpose: Verify all buttons work correctly in different states

Test Steps:

  • Test "Open Settings" button opens the options page
  • Test "Clear Cache" button clears cache and shows success message
  • Verify "Clear Cache" is disabled during analysis
  • Verify "Clear Cache" shows loading state while clearing
  • Verify buttons have proper focus indicators (Tab key)
  • Verify buttons can be activated with Enter/Space keys
